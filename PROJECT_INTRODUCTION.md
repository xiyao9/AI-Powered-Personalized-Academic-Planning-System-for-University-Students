# AI 大学生个性化学业规划系统 - 项目介绍

## 一、项目概述

### 1.1 项目名称
AI 大学生个性化学业规划助手

### 1.2 项目背景与痛点分析

当前在校大学生在学业规划方面普遍面临以下核心痛点：

1. **缺乏系统规划**：大学四年没有清晰的学业、考研、就业、考公规划路径，完全不知道从哪里开始、应该朝哪个方向努力
2. **学习路径缺失**：想自学提升，但没有系统性的学习路径指导，盲目摸索，效率低下
3. **信息过载困扰**：网上网课、学习资源极其丰富但质量参差不齐，学生难以分辨哪个适合自己
4. **方案缺乏个性化**：网上的规划模板千篇一律，不贴合学生自身专业、年级和发展方向
5. **执行力不足**：做了规划缺乏监督和动态调整机制，容易半途而废

### 1.3 解决方案

本项目利用 AI 大语言模型技术，为在校大学生提供一站式个性化学业规划服务。系统通过收集学生的年级、专业、发展方向、短板和兴趣等信息，自动生成详细的四年规划方案，并提供学习资源推荐、进度跟踪打卡、智能问答等配套功能，形成"规划 → 执行 → 反馈 → 调整"的完整闭环。

## 二、核心功能模块

### 2.1 用户信息录入
- 支持学生注册并完善个人信息
- 选择年级（大一至大四）、专业、未来发展方向（考研 / 就业 / 考公 / 留学 / 暂未确定）
- 填写自身短板和兴趣倾向，为 AI 生成个性化方案提供依据

### 2.2 AI 专属规划生成
- 基于大模型 API 自动生成分学年、分学期的详细规划
- 规划内容包括：课程学习重点、必考证清单、实习建议、学习任务分解、时间管理建议
- 支持重新生成规划，结合最新情况进行动态调整

### 2.3 智能学习资源推荐
- 根据专业和发展方向，智能推荐高质量网课、参考书籍、实用工具和刷题资源
- 帮助学生避开劣质课程，精准匹配适合自身的学习资源
- 支持 AI 推荐与数据库资源相结合

### 2.4 进度打卡与跟踪
- 支持每日任务打卡，记录完成情况和进度备注
- 自动计算连续打卡天数、总打卡天数等统计数据
- AI 根据打卡内容给予针对性反馈和鼓励建议
- 可视化展示打卡历史

### 2.5 AI 智能问答
- 学生可随时与 AI 对话，咨询专业学习方法、考研备考、就业指导等问题
- AI 结合学生个人信息提供针对性建议
- 保留完整的对话历史记录，方便回顾

### 2.6 后台管理系统
- 管理员认证登录
- 专业模板管理（增删查）
- 学习资源库管理（增删查）
- 系统数据概览看板

## 三、技术架构

### 3.1 总体架构

```
┌─────────────────────────────────────────┐
│              前端（Vue 3 SPA）            │
│   index.html + main.css + main.js       │
│   · 用户注册  · 规划展示  · 资源推荐      │
│   · 打卡记录  · AI 问答  · 后台管理       │
└──────────────┬──────────────────────────┘
               │ HTTP / RESTful API
┌──────────────▼──────────────────────────┐
│            后端（FastAPI）                │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐ │
│  │ 用户模块 │ │ 规划模块 │ │ 资源模块   │ │
│  └─────────┘ └─────────┘ └───────────┘ │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐ │
│  │ 打卡模块 │ │ AI 问答  │ │ 后台管理   │ │
│  └─────────┘ └─────────┘ └───────────┘ │
└──────┬───────────────────────┬──────────┘
       │                       │
┌──────▼──────┐       ┌────────▼─────────┐
│   MySQL     │       │  AI 大模型 API    │
│  数据库      │       │  (Qwen/通义千问)  │
└─────────────┘       └──────────────────┘
```

### 3.2 前端技术栈

| 技术 | 说明 |
|------|------|
| Vue 3 | 响应式前端框架（CDN 引入） |
| 自定义 CSS | 渐变色主题，响应式布局 |
| Fetch API | 与后端 RESTful API 通信 |

### 3.3 后端技术栈

| 技术 | 说明 |
|------|------|
| FastAPI | 高性能 Python Web 框架 |
| SQLAlchemy 2.0 | Python ORM 框架 |
| Pydantic v2 | 数据验证与序列化 |
| Uvicorn | ASGI 服务器 |
| PyMySQL | MySQL 驱动 |

### 3.4 AI 能力

| 技术 | 说明 |
|------|------|
| OpenAI 兼容 API | 兼容通义千问、DeepSeek、智谱等大模型 |
| 模拟数据模式 | 无 API Key 时内置丰富预设响应 |

### 3.5 数据库设计

系统共 7 张核心数据表：

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| users | 用户信息表 | 姓名、年级、专业、方向、短板、兴趣 |
| planning_schemes | 规划方案表 | 用户ID、学年规划、学期规划、考证清单、实习建议 |
| resources | 学习资源表 | 资源名称、类型、适用专业、描述、链接 |
| checkin_records | 打卡记录表 | 用户ID、日期、完成任务、进度备注、AI 反馈 |
| ai_dialogs | AI 对话表 | 用户ID、问题、回答 |
| major_templates | 专业模板表 | 专业名称、核心课程、推荐证书、职业路径 |
| admins | 管理员表 | 用户名、密码 |

## 四、项目创新点

1. **AI 驱动的个性化方案**：摒弃传统固定模板，AI 根据每位学生的专业、年级、方向、短板、兴趣生成独一无二的规划
2. **完整闭环机制**："规划生成 → 任务执行 → 打卡反馈 → 动态调整"形成完整的自我管理闭环
3. **智能资源整合**：按专业和发展方向精准推荐高质量学习资源，解决信息筛选难题
4. **24 小时在线顾问**：AI 智能问答满足随时随地的学业咨询需求
5. **零门槛部署**：技术选型贴合学生群体能力水平，无需复杂配置即可运行
6. **双模式运行**：支持真实 AI API 和内置模拟数据两种模式，适配不同使用场景

## 八、开发环境与部署

详见 [DEPLOYMENT_GUIDE.md](DEPLOYMENT_GUIDE.md) 文档。核心步骤：

1. 安装 Python 3.8+ 和 MySQL 8.0+
2. `pip install -r requirements.txt`
3. `python init_db.py` 初始化数据库
4. `python run_server.py` 启动服务
5. 浏览器访问 http://localhost:8000
